Producers

A producer is any application that publishes events to a Kafka topic. Producers are the source of data in a Kafka system.

Producer Basics

Here's a minimal producer in Java (confluent Kafka client):

Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer",   "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

producer.send(new ProducerRecord<>("orders", "order-42", "{\"item\":\"laptop\"}"));

producer.close();

Key Producer Settings

Acknowledgements (acks)

Controls how many partition replicas must acknowledge a write before it's considered successful:

acks Behavior Durability
0 Fire-and-forget. No acknowledgment. ❌ None
1 Leader acknowledges (default). ⚠️ Partial
all All in-sync replicas acknowledge. ✅ Strongest
Recommendation Use acks=all in production unless you have a specific reason not to. The latency cost is small (one extra RTT to followers), and the durability gain is significant.

Batching

For throughput, producers batch events before sending them to the broker:

  • linger.ms — how long to wait for more events before sending a batch (default 0)
  • batch.size — max bytes for a single batch (default 16 KB)
props.put("linger.ms",  5);    // wait up to 5ms to fill a batch
props.put("batch.size", 32768); // 32 KB batches

Compression

Compress producer data to reduce network and storage usage. gzip and snappy are common choices. 

props.put("compression.type", "snappy");

Idempotent Producers

Set enable.idempotence=true to prevent duplicate events caused by network retries. The producer assigns a unique sequence number to each event, and the broker deduplicates them. This gives exactly-once semantics for producer-to-broker writes. 

props.put("enable.idempotence", true);
// Also recommended for idempotent mode:
props.put("acks", "all");

Async Send with Callback

producer.send(
  new ProducerRecord<>("orders", "order-42", event),
  (metadata, exception) -> {
    if (exception != null) {
      log.error("Send failed", exception);
    } else {
      log.info("Sent to partition {} at offset {}",
        metadata.partition(), metadata.offset());
    }
  }
);

Key Takeaway

Configure your producer for your durability vs. latency trade-off. Use idempotent producers for critical data paths. Batch and compress for throughput.
